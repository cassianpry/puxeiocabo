Componentes de domínio — o que construir
FighterSearchCombobox
shadcn Popover + Command. Input dispara GET /fighters?q= com useDebounce(300ms) via TanStack Query (enabled: q.length > 1). Sem estado local de fetch — tudo no queryClient. Nunca modificar Command do shadcn.
ReportForm
react-hook-form + zod para validação client-side. Upload direto ao backend com fetch (credentials:'include'). Progresso de upload via XHR ou fetch stream. Usa shadcn Form, Input, Textarea — sem modificações.
FighterCard
shadcn Card com foto, nome, record W/L/D, badges de categoria. Link para /_auth/fighters/$id via TanStack Router Link.
PageShell
Wrapper de layout interno ao _auth.tsx. Contém Sidebar colapsável + área de conteúdo. Sidebar usa shadcn Sheet em mobile, sidebar fixa em desktop.
DataTable (relatórios)
shadcn Table — sem lib externa. Colunas: data, lutadores, status, ações. Paginação via TanStack Query (offset/limit na URL como searchParams do TanStack Router).
StatusBadge
shadcn Badge com variantes mapeadas para status da API. Variant map em um objeto const — não criar variantes customizadas no shadcn.
Regras de implementação — o que não tocar
shadcn/ui: nenhum arquivo em components/ui/ é editado. Customização via CSS variables em globals.css e via className prop nos pontos de uso.
TanStack Router: nenhuma modificação no core. Guards implementados via beforeLoad nos layouts (_auth.tsx). Redirect para /login se não autenticado.
TanStack Query: nenhuma modificação no core. Configuração apenas no queryClient.ts (staleTime, retry, etc.).
Dark theme forçado: document.documentElement.classList.add('dark') em main.tsx. Sem toggle, sem localStorage de tema. Tailwind config: darkMode: 'class'.
Credenciais: todo fetch via lib/api.ts com credentials: 'include'. Nunca passar token em header manualmente.
Autocomplete: debounce no hook, não no componente. useQuery com enabled: query.length > 1 para evitar requests desnecessários.
globals.css — estrutura mínima
/* 1 */ @tailwind base; @tailwind components; @tailwind utilities;
/* 2 */ :root { /* tokens shadcn no modo dark */ }
/* 3 */ .dark { --background: 240 10% 3.9%; --foreground: 0 0% 98%; /* … todos os tokens */ }
/* 4 */ html { @apply dark; } /* força dark globalmente */
Alternativa: adicionar className="dark" direto no <html> via index.html ou em main.tsx antes do render.

darkMode: 'class',
content: ['./index.html', './src/**/*.{ts,tsx}'],
theme.extend.colors: { /* mapear vars CSS do shadcn */ }
plugins: [require('tailwindcss-animate')] /* usado pelo shadcn */
